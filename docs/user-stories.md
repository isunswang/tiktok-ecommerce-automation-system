## 1. 用户认证与权限

### US-01: 用户注册
- **作为** 新用户
- **我想要** 通过邮箱和密码注册账号
- **以便** 我可以登录系统使用跨境电商运营功能
- **验收标准**:
  1. 提供有效的邮箱和符合强度要求的密码（最少8位，包含字母和数字）可成功注册
  2. 重复邮箱注册时返回明确的错误提示
  3. 注册成功后返回JWT Token
- **优先级**: P0

### US-02: 用户登录
- **作为** 已注册用户
- **我想要** 使用邮箱和密码登录系统
- **以便** 我可以访问系统功能
- **验收标准**:
  1. 正确的邮箱和密码登录成功，返回access_token和refresh_token
  2. 错误的密码登录失败，返回401错误
  3. 连续5次登录失败后锁定账号15分钟
- **优先级**: P0

### US-03: Token刷新
- **作为** 已登录用户
- **我想要** 在access_token过期前通过refresh_token获取新token
- **以便** 我无需重新登录即可继续使用系统
- **验收标准**:
  1. 使用有效的refresh_token成功获取新的access_token
  2. 过期的refresh_token返回401错误并要求重新登录
- **优先级**: P0

### US-04: 获取当前用户信息
- **作为** 已登录用户
- **我想要** 查看我的个人信息和权限
- **以便** 我可以了解自己在系统中的角色和权限范围
- **验收标准**:
  1. 使用有效token获取当前用户的邮箱、姓名、角色信息
  2. 返回信息包含用户角色和权限列表
- **优先级**: P0

### US-05: 角色权限管理
- **作为** 管理员
- **我想要** 创建和管理系统角色及其权限
- **以便** 不同角色的用户只能访问其职责范围内的功能
- **验收标准**:
  1. 管理员可以创建自定义角色并分配模块级权限
  2. 系统预设管理员、运营人员、客服人员、财务人员四种角色
  3. 用户角色变更后立即生效
- **优先级**: P0

- **优先级**: P0
